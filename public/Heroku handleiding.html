Heroku Handleiding

Inleiding tot Heroku

Heroku is een Platform-as-a-Service (PaaS) dat ontwikkelaars in staat stelt om applicaties snel en eenvoudig te bouwen, te deployen en te beheren in de cloud. Het is populair vanwege de eenvoudige integratie met Git en de mogelijkheid om applicaties zonder veel moeite op schaal te draaien.

Installatie van Heroku CLI

  1. Download Heroku CLI: Ga naar de Heroku CLI downloadpagina en download de versie voor jouw besturingssysteem.
  2. Installeer Heroku CLI: Volg de installatie-instructies van het installatieprogramma.
  3. Controleer de installatie: Open de terminal of command prompt en typ:
    heroku --version

Een Heroku-app Aanmaken

  1. Log in op Heroku: Voer het volgende commando uit om in te loggen bij Heroku via de CLI:
    heroku login
  2. Maak een nieuwe Heroku-app: Ga naar je projectmap en voer het volgende commando uit:
    heroku create naam-van-je-app

Een Applicatie Deployen naar Heroku

  1. Initialiseer een Git-repository: Als je project nog niet is geïnitialiseerd als Git-repository, doe dat dan met:
    git init
  2. Voeg je bestanden toe aan Git:
    git add .
  3. Commit je wijzigingen:
    git commit -m "Initial commit"
  4. Deploy naar Heroku: Push je code naar Heroku met het volgende commando:
    git push heroku main

Veelvoorkomende Problemen en Oplossingen

  1. Port Binding Probleem (R10 Error): Heroku vereist dat je applicatie luistert op een door Heroku toegewezen poort. Zorg ervoor dat je de poort instelt via de omgevingsvariabele $PORT.
    import os
    port = int(os.environ.get("PORT", 5000))
    app.run(host="0.0.0.0", port=port)
  2. Compatibiliteitsproblemen met Afhankelijkheden: Soms kunnen bepaalde versies van bibliotheken zoals Numpy en Pandas niet goed samenwerken. Dit kan leiden tot crashes.
    requirements.txt file update with specific versions
  3. Heroku Dyno Limieten: Gratis dyno's slapen na 30 minuten inactiviteit en hebben een maandelijkse limiet op het aantal uren dat ze actief kunnen zijn. Overweeg een upgrade naar betaalde dyno's als je applicatie continu beschikbaar moet zijn.

Heroku Logs en Debugging

  1. Bekijk de Heroku logs:
    heroku logs --tail
  2. Herstart je app: Soms kan een herstart van je dyno helpen om problemen op te lossen.
    heroku restart

Beheer van Omgevingsvariabelen

  1. Instellen van een omgevingsvariabele:
    heroku config:set VARIABLE_NAME=value
  2. Bekijken van omgevingsvariabelen:
    heroku config

Kostenbeheer op Heroku

  1. Controleer je dyno-gebruik:
    heroku ps
  2. Upgrade naar een betaalde dyno: als je meer resources nodig hebt:
    heroku ps:scale web=1:Standard-1X
  3. Beheer add-ons: Sommige add-ons kunnen kosten met zich meebrengen. Beheer deze zorgvuldig via het Heroku-dashboard of de CLI.

Samenvatting

Heroku is een krachtig en gebruiksvriendelijk platform voor het hosten van webapplicaties. Met de bovenstaande commando's en voorbeelden kun je snel aan de slag om je applicaties te deployen, beheren en debuggen. Houd je kosten in de gaten door slim gebruik te maken van dyno's en add-ons, en zorg ervoor dat je omgevingsvariabelen correct instelt voor een efficiënte configuratie.

Met deze handleiding heb je de tools in handen om Heroku effectief te gebruiken in je ontwikkelproces.
