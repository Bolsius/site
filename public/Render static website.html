Handleiding voor het Uploaden naar Render

Stap 1: Initialiseer een Git Repository

  1. Navigeer naar je projectmap:
    cd pad/naar/jouw/project
  2. Initialiseer Git:
    git init
  3. Voeg bestanden toe aan Git:
    git add .
  4. Commit de wijzigingen:
    git commit -m "Initial commit"

Stap 2: Koppel met GitHub en Push

  1. Maak een GitHub repository en voeg de remote toe:
    git remote add origin https://github.com/jouw-gebruikersnaam/jouw-repository.git
  2. Push de bestanden naar GitHub:
    git push -u origin main

Stap 3: Maak een Render Account en Koppel met GitHub

  1. Ga naar render.com en maak een account aan.
  2. Koppel je GitHub account door in te loggen op Render en de permissies toe te staan.

Stap 4: Maak een nieuwe Web Service

  1. Na het inloggen op Render, klik op “New” en kies “Web Service”.
  2. Selecteer de repository die je wilt deployen of voer de Git URL in. Kies daarna “Continue”.

Stap 5: Configureer de Web Service

  1. Geef je service een naam (bijv. “Handleiding”).
  2. Kies de juiste build-instellingen (bijv. public voor HTML). Vul de root directory in (bijv. ./public).

Stap 6: Voltooi de Deploy

  1. Render zal nu de bestanden builden en je site live zetten. Volg de logs op Render om de voortgang te volgen.
  2. Zodra de deploy is voltooid, kun je je website openen via de gegenereerde URL (bijv. https://jouw-website.onrender.com).

Veelvoorkomende Problemen

Als je een "404 Not Found" foutmelding krijgt, controleer of je een index.html bestand hebt geüpload en of dit zich in de juiste map bevindt.

Handmatig een nieuwe deploy forceren via de terminal

Als Render je wijzigingen niet automatisch oppikt, volg dan deze stappen:

  1. Controleer de status van je repository om te zien of er onopgeslagen bestanden zijn:
    git status
  2. Voeg onopgeslagen bestanden toe:
    git add .
  3. Commit de wijzigingen:
    git commit -m "Beschrijving van de wijziging"
  4. Push de wijzigingen naar GitHub:
    git push origin main
  5. Als Render je wijzigingen niet oppikt, forceer een nieuwe commit om een deploy te triggeren:
    git commit --allow-empty -m "Trigger rebuild"
