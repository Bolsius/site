Git Handleiding

Inleiding tot Git

Git is een gedistribueerd versiebeheersysteem dat je helpt om veranderingen in bestanden bij te houden. Het wordt vaak gebruikt voor het beheren van broncode in softwareontwikkeling, maar kan ook worden gebruikt om de geschiedenis van wijzigingen in elk bestand te beheren.

Basisconcepten van Git

  • Repository (Repo): Een opslagplaats die alle bestanden en mappen bevat die Git volgt, inclusief een volledige geschiedenis van alle veranderingen.
  • Commit: Een momentopname van je project op een bepaald moment. Elke commit bevat een unieke ID, een beschrijving, en informatie over welke bestanden zijn veranderd.
  • Branch: Een parallelle versie van je repository. Elke branch kan onafhankelijk van de andere branches evolueren. De main of master branch is vaak de hoofdbranch.
  • Remote: Een externe versie van je repository, bijvoorbeeld op GitHub of Heroku, waarmee je je project kunt delen en samenwerken met anderen.

Installatie van Git

  1. Download Git: Ga naar Git-website en download de versie voor jouw besturingssysteem (Windows, macOS, Linux).
  2. Installeer Git: Volg de installatie-instructies in het installatieprogramma.
  3. Controleer de installatie: Open de terminal of command prompt en typ:
    git --version

Initialiseren van een Git-repository

Een Git-repository initialiseren betekent dat je Git vertelt om de huidige map te gaan volgen en beheren.

  1. Navigeer naar je projectmap: Gebruik het cd-commando in de terminal om naar de map te gaan waarin je project zich bevindt.
  2. Initialiseer Git: Typ het volgende commando:
    git init

Bestanden toevoegen aan Git

Wanneer je nieuwe bestanden maakt of bestaande bestanden wijzigt, moet je Git vertellen om die wijzigingen te volgen.

  1. Voeg een specifiek bestand toe:
    git add bestandsnaam.ext
  2. Voeg alle bestanden in één keer toe:
    git add .

Een Commit maken

Een commit is een momentopname van je project op een bepaald moment.

  1. Maak een commit: Typ:
    git commit -m "Beschrijvend bericht"

Status van je Repository Bekijken

Je kunt de huidige status van je project bekijken, inclusief welke bestanden zijn gewijzigd en welke nog niet zijn gecommit.

git status

Bestanden die door Git worden gevolgd bekijken

Je kunt een lijst bekijken van alle bestanden die op dit moment door Git worden gevolgd.

git ls-files

Toegang tot Logs en Versiegeschiedenis

Je kunt de geschiedenis van commits en andere Git-acties bekijken.

  1. Gebruik git log:
    git log
  2. Details van een specifieke commit bekijken:
    git show commit-id

Bestanden in Submappen toevoegen

Bestanden in submappen kunnen eenvoudig worden toegevoegd door het volledige pad op te geven:

git add templates/index.html

Verwijderde Bestanden Herstellen

Als je per ongeluk bestanden hebt verwijderd die door Git worden gevolgd, kun je deze herstellen.

  1. Controleer welke bestanden zijn verwijderd: Gebruik git status om te zien welke bestanden zijn verwijderd.
    git status
  2. Herstel de verwijderde bestanden: Gebruik git restore om de verwijderde bestanden te herstellen.
    git restore path/naar/verwijderd/bestand
  3. Commit de wijzigingen: Als je de verwijderingen wilt doorvoeren, commit dan de wijzigingen.
    git add -u && git commit -m "Verwijder Sandbox map"

Wijzigingen Pushen naar Heroku

Nadat je wijzigingen hebt aangebracht, kun je deze naar Heroku pushen om je live-app bij te werken.

  1. Controleer de status: Gebruik git status om te controleren welke wijzigingen klaar zijn om gecommit en gepusht te worden.
    git status
  2. Commit de wijzigingen: Voeg de wijzigingen toe en commit ze.
    git add -u && git commit -m "Jouw beschrijvende bericht"
  3. Push de wijzigingen naar Heroku: Gebruik het volgende commando om de wijzigingen naar Heroku te pushen.
    git push heroku main
  4. Bekijk de logs (optioneel): Controleer de Heroku logs om te zien of alles correct is gedeployed.
    heroku logs --tail

Belangrijkste Git-commando's

Hier is een lijst van de 25 belangrijkste Git-commando's die je vaak zult gebruiken.

  1. Git Init: Initialiseer een nieuwe Git-repository.
    git init
  2. Git Clone: Maak een kopie van een bestaande repository.
    git clone URL
  3. Git Status: Bekijk de huidige status van de repository.
    git status
  4. Git Add: Voeg wijzigingen toe aan de staging area.
    git add bestandsnaam.ext
  5. Git Commit: Sla de staged wijzigingen op in de repository.
    git commit -m "Beschrijvend bericht"
  6. Git Push: Verstuur de wijzigingen naar een remote repository.
    git push origin main
  7. Git Pull: Haal en integreer wijzigingen van een remote repository.
    git pull origin main
  8. Git Branch: Maak een nieuwe branch of bekijk bestaande branches.
    git branch
  9. Git Checkout: Wissel van branch of herstel bestanden.
    git checkout branchnaam
  10. Git Merge: Voeg wijzigingen van een andere branch samen in de huidige branch.
    git merge branchnaam
  11. Git Log: Bekijk de commitgeschiedenis.
    git log
  12. Git Reset: Zet de huidige HEAD terug naar een eerdere commit.
    git reset --hard commit-id
  13. Git Revert: Maak een nieuwe commit die de wijzigingen van een eerdere commit ongedaan maakt.
    git revert commit-id
  14. Git Stash: Sla wijzigingen tijdelijk op en herstel de werkdirectory naar een schone staat.
    git stash
  15. Git Stash Pop: Haal opgeslagen wijzigingen op uit de stash en pas ze toe.
    git stash pop
  16. Git Fetch: Haal wijzigingen van de remote repository zonder ze te mergen.
    git fetch
  17. Git Remote: Beheer je remote repositories.
    git remote -v
  18. Git Tag: Markeer specifieke punten in de geschiedenis als belangrijk.
    git tag -a v1.0 -m "Versie 1.0"
  19. Git Diff: Toon de verschillen tussen de huidige status en de laatste commit.
    git diff
  20. Git Blame: Toon welke wijzigingen door welke auteur zijn aangebracht in een bestand.
    git blame bestandsnaam.ext
  21. Git Show: Toon gedetailleerde informatie over een specifieke commit.
    git show commit-id
  22. Git Rm: Verwijder een bestand uit de werkdirectory en het staging area.
    git rm bestandsnaam.ext
  23. Git Clean: Verwijder ongetrackte bestanden uit de werkdirectory.
    git clean -f

Veelgestelde Vragen en Problemen

  • Wat betekent de output van git status?
    De output laat zien of je in de main branch werkt, en of er ongetrackte bestanden zijn die nog niet aan Git zijn toegevoegd. Dit betekent dat er bestanden in je map staan die Git nog niet volgt.
  • Hoe kan ik zien welke mappen Git volgt?
    Gebruik git ls-files om een lijst te zien van alle bestanden die Git volgt, inclusief bestanden in submappen.
  • Wat gebeurt er als ik git add gebruik op een bestand in een submap?
    Als je git add gebruikt op een bestand in een submap (bijv. git add templates/index.html), dan wordt dat bestand correct toegevoegd en gevolgd door Git.
  • Wat als ik tree wil gebruiken op Windows?
    Op Windows kun je het ingebouwde tree /F commando gebruiken om een boomstructuur van je bestanden en mappen te bekijken.
  • Worden alle bestanden getrackt door Git?
    Alleen de bestanden die worden weergegeven door git ls-files worden getrackt door Git. Ongetrackte bestanden worden niet opgenomen in commits totdat je ze toevoegt met git add.

Gebruik van Git met Remote Repositories

Git kan samenwerken met online diensten zoals GitHub of Heroku om je project op te slaan in de cloud en samenwerking mogelijk te maken.

  1. Remote toevoegen:
    git remote add origin https://github.com/gebruikersnaam/repositorynaam.git
  2. Code pushen naar remote:
    git push origin main

Samenvatting

Door deze basiscommando's en concepten te begrijpen, kun je effectief gebruikmaken van Git in je eigen projecten.
