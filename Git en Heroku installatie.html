Heroku Deployment Stappen

1. Voorbereiding van het Project

requirements.txt bestand

Dit bestand is gegenereerd op basis van de Python-pakketten die in je project worden gebruikt.

pip freeze > requirements.txt

In jouw geval bevatte het bestand:

Flask==2.3.2
pandas==1.5.3
openpyxl==3.1.2
xlsxwriter==3.0.9

Procfile bestand

Dit bestand vertelt Heroku welke command moet worden uitgevoerd om je applicatie te starten.

web: python app.py

Git-repository initialiseren

We hebben ervoor gezorgd dat je project een Git-repository is, zodat je het naar Heroku kunt deployen.

git init
git add .
git commit -m "Initial commit"

2. Heroku Instellen

Heroku CLI installeren

Dit was nodig om Heroku-commando's in je terminal te kunnen gebruiken.

Heroku CLI Installatie

Heroku App aanmaken

We hebben een Heroku-app genaamd flashcard-unitasfam gemaakt.

heroku create flashcard-unitasfam

Heroku Remote Repository toevoegen

We hebben Heroku toegevoegd als een remote repository in je Git-project.

git remote add heroku https://git.heroku.com/flashcard-unitasfam.git

3. Deployen naar Heroku

Code pushen naar Heroku

We hebben je code naar Heroku gepusht om je app te deployen.

git push heroku main

Problemen met Gebruikersinstellingen in Git

We hebben je gebruikersnaam en e-mailadres ingesteld om te voorkomen dat Git vroeg wie je bent tijdens het maken van commits.

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Deploymentproces bij Heroku

Het deploymentproces kan soms langer duren, vooral als Heroku pakketten moet installeren en bouwen (zoals pandas). Dit is normaal en vereist enige geduld.

4. App Openen en Logs Controleren

App openen

Nadat de deployment succesvol was, kun je je app openen in een webbrowser.

heroku open

Logs controleren

Als je problemen tegenkomt, kun je de Heroku logs controleren om te zien wat er misgaat.

heroku logs --tail
